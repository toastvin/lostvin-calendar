# 달력 프린트 웹서비스 계획안 v3 (가칭)

## 1. 프로젝트 개요

- **프로젝트명**: 미정
  - 후보: `CalPrint`, `PrintCal`, `PaperCalendar`, `달력공방`
  - 도메인: `calprint.kr`, `printcal.kr`, `papercalendar.xyz` 등
- **기술 스택**: Next.js + React + TypeScript
- **홈페이지 목적**
  사용자가 **직접 프린트해서 쓰는 달력**을 쉽고 예쁘게 만들 수 있는 웹서비스.
  로그인 없이도 빠르게 달력을 만들고, 출력하거나 PDF로 저장할 수 있는 조용한 도구를 지향한다.
- **수익 목적**: 일정 수준의 방문자 유입 후
  **Google AdSense(구글 광고)** 를 통한 광고 수익 확보

## 2. 서비스 컨셉 & 핵심 가치

### 컨셉 한 줄 요약
> "책상 위에 올려둘 종이 달력을, 몇 번의 클릭으로 만들어주는 조용한 공방"

### 핵심 가치

1. **프린트 친화성**
   - 집/사무실 프린터로 바로 출력해도 레이아웃이 깨지지 않는 구조
   - 흑백 출력, 잉크 절약 모드, 여백 최적화 등 현실적인 인쇄 환경 고려
   - 🆕 **Eco 모드 (잉크 절약)**: 검은색 → 진한 회색(#555), 얇은 선, 점선 테두리

2. **간단함**
   - 3~4단계 내에 달력 생성 완료
   - 회원가입, 복잡한 계정 시스템 없이 누구나 바로 사용 가능
   - 🆕 **브라우저 언어 자동 감지**: 한국어/영어 자동 전환

3. **다국가 공휴일 지원**
   - 여러 나라 공휴일을 한 장의 달력에 겹쳐 볼 수 있음
   - 해외 지사, 유학생, 글로벌 팀에 유용
   - 🆕 **공휴일 데이터 출처 명시 + 연도별 페이지** (SEO 강화)

4. **종이 도구 감성**
   - 화면보다 **인쇄 후 모습**이 더 중요
   - 단정한 타이포, 여백, 종이 느낌을 살린 디자인

## 3. 타깃 사용자 & 사용 시나리오

### 3.1. 타깃 사용자

- **학원/학교 담당자**
  - 학사 일정, 시험 일정, 방학 일정을 한 장에 정리해 배포하고 싶다.
  - 🆕 상단에 "○○학원 2025년 시간표" 같은 큰 제목을 넣고 싶다.
- **스타트업/소규모 팀장**
  - 프로젝트 마감, 릴리즈 일정, 휴가 일정을 함께 볼 수 있는 달력이 필요하다.
  - 🆕 타임라인 형태로 3~6개월을 가로로 보고 싶다.
- **프리랜서/개인**
  - 개인 플래너, 공부 계획, 습관 기록용 달력을 프린트해서 쓰고 싶다.
  - 🆕 체크박스/점찍기 형태의 습관 추적용 달력이 필요하다.
- **해외 생활자/글로벌 팀**
  - 한국 + 거주국 공휴일을 동시에 보고 싶다.
- **🆕 육아맘/요리 좋아하는 사람**
  - 이유식/식단표를 달력 형태로 관리하고 싶다.

### 3.2. 주요 사용 시나리오

1. 새해가 다가올 때, **연간 달력**을 한 번에 뽑아서 벽에 붙이고 싶다.
2. 3~6개월짜리 프로젝트 타임라인을 **가로형 달력**에 정리하고 싶다.
3. 한국 + 미국 공휴일이 모두 표시된 달력을 만들어 팀과 공유하고 싶다.
4. 개인 공부 계획을 세우기 위해 **월별 달력 + 넓은 메모 공간**이 있는 양식을 만들고 싶다.
5. 🆕 모바일에서 설정 완료 → PC에서 같은 링크 열어서 바로 인쇄하고 싶다.
6. 🆕 잉크 절약해서 프린트하고 싶다.

## 4. 제공 기능 (MVP 기준)

### 4.1. 달력 유형

**1차 버전(MVP)에서 제공할 달력 타입**

#### 기본 유형
- **연간 달력**
  - 한 페이지에 12개월 표시
  - 벽걸이용, 팀/조직 전체 일정 조망용
  - 🆕 **A4 1장 vs A4 2장(A3 크기) 선택 옵션**
- **월간 달력**
  - 1페이지 1개월 기준
  - "이번 달 일정" 프린트용
  - 🆕 **상단 큰 제목란 옵션** (예: "2025년 3월 - ○○학원 시간표")
- **3개월/6개월 묶음 달력**
  - 프로젝트용 중기 일정 보기 좋게
  - 🆕 **가로형 타임라인 스타일 추가**

#### 🆕 목적별 프리셋 (니치 타겟팅 + SEO)
사용자가 바로 쓸 수 있는 "용도별 템플릿" 제공

1. **기본 달력**
   - 일반적인 연간/월간 달력

2. **습관 추적용 (Habit Tracker)**
   - 날짜 밑에 작은 체크박스 3개 배치
   - 검색 키워드: "습관 달력", "habit tracker printable"

3. **식단표/이유식 달력**
   - 칸이 크고 메모 공간 위주
   - 검색 키워드: "이유식 식단표", "식단 달력"

4. **공부/시험 D-Day 달력**
   - 하단에 D-Day 카운트나 목표 적는 칸 배치
   - 검색 키워드: "수능 달력", "D-Day 달력"

5. **교대근무표 (Ver.2 이후 고도화)**
   - 3교대, 4교대 패턴 입력하면 색칠
   - 검색 키워드: "근무표 양식"

### 4.2. 기본 설정 옵션

- **기간/연도 설정**
  - 🆕 **"올해(2025)", "내년(2026)" 퀵 버튼** (상단에 크게 배치)
  - 연도 수동 선택: 2020~2030
  - 표시 기간: 전체 연도 / 특정 월 범위

- **국가별 공휴일**
  - 1차 지원 국가:
    - 한국(KR)
    - 미국(US)
    - 일본(JP)
    - 중국(CN)
  - 여러 국가를 동시에 선택할 수 있음 (예: KR + US)
  - 🆕 **공휴일 색상 커스텀** (국가별 색상 선택 가능)
  - 🆕 **공휴일 데이터 출처 페이지**: `/holidays/2026/kr` 형태로 정적 페이지 생성 (SEO)

- **요일/주말 설정**
  - 주 시작 요일: 월요일/일요일 선택
  - 주말 강조 여부: 회색 배경, 테두리 등
  - 🆕 **오늘 날짜 하이라이트**: 미리보기와 PDF에 오늘 날짜 연한 노란색 원 표시

- **종이 & 방향**
  - 용지 크기: A4 / A3 (기본은 A4)
  - 방향: 세로 / 가로

- **음력 표시 (옵션)**
  - 음력 날짜 작게 병기
  - 주요 명절(설날, 추석 등) 자동 표시
  - 한국/중국/일본 시장 대응

- **주차 번호 표시 (옵션)**
  - 연간 주차 번호 (Week 1~52) 표시
  - ISO 주차 표준 지원
  - 프로젝트 관리/학사 일정용

### 4.3. 스타일 옵션 (MVP용 최소셋)

- **테마**
  - 기본 흑백 테마 (프린트 최적화)
  - 🆕 **Eco 모드 (잉크 절약)**
    - 검은색(#000) → 진한 회색(#555)
    - 선 두께 얇게, 테두리 점선으로 변경
    - 토너 소모량 급감 + 사용자 만족도 향상
  - 다크모드 (화면 설정용, 프린트는 항상 흰색 배경)

- **폰트**
  - 2~3개 선택지 제공
    - 정갈한 명조체 (예: Gowun Batang)
    - 깔끔한 고딕체
    - (선택) 손글씨 느낌 폰트

- **날짜/메모 비율**
  - 기본형: 날짜칸 + 적당한 메모 공간
  - 메모공간 확대형: 식단표/습관 추적용

- **🆕 작은 워터마크 (옵션)**
  - 달력 하단에 "made with [서비스명]" 작게 표시
  - 사용자가 끌 수 있게 옵션 제공

### 4.4. 간단한 메모/이벤트 입력 (MVP 포함)

- **특정 날짜에 한 줄 텍스트 입력**
  - 날짜 클릭 → 텍스트 입력 → 달력에 표시
  - 🆕 **LocalStorage에 저장** (새로고침해도 유지)
  - 🆕 **마지막 설정 기억하기**: 주 시작 요일, 폰트, 국가 등 자동 저장 후 재방문 시 불러오기
- **기본 기능만 제공**
  - 색상/카테고리 구분은 Ver.2 이후
  - CSV 업로드는 향후 확장

### 4.5. URL 공유 기능 (MVP 필수) ⭐⭐⭐

**중요도 최상위 - 바이럴 + SEO + 재방문 동시 해결**

- **현재 설정을 URL로 완벽 공유**
  - 예: `?year=2025&type=yearly&countries=KR,US&start=mon&paper=A4&orient=landscape&eco=true`
  - URL 쿼리 파라미터로 모든 설정(연도, 국가, 레이아웃, 스타일, 메모 내용 등) 인코딩
  - Next.js 특성상 구현 난이도 매우 낮음

- **공유 시나리오**
  - **모바일 → PC 이어하기**: 모바일에서 설정 완료 → 링크 복사 → PC에서 열어 바로 인쇄
  - **팀 공유**: 카카오톡/슬랙으로 링크 공유 → 팀원들이 동일한 달력 즉시 생성
  - **SNS 공유**: "이렇게 예쁜 달력 만들었어요!" + 링크

- **🆕 공유 버튼 강화**
  - "링크 복사하기" 버튼
  - 🆕 **"카카오톡으로 나에게 보내기" 버튼** (모바일 → PC 이어하기 핵심)
  - 🆕 **모바일 전용: "PC에서 열기 위해 링크 복사" 강조**

### 4.6. 출력 & 다운로드 ⭐⭐⭐

**기술적 의사결정 중요**

#### 🆕 PDF 생성 방식 (최우선 검토 필요)

**문제점**: 브라우저 기본 인쇄(`window.print()`)는 사용자의 브라우저(Chrome, Safari), OS, 프린터 설정에 따라 여백이 제각각으로 나옴. 화면에서 본 예쁜 여백이 실제로는 잘릴 수 있음.

**해결책**:
- `@react-pdf/renderer` 같은 라이브러리를 사용하여 **클라이언트 사이드에서 PDF 직접 렌더링**
- 사용자가 어떤 브라우저를 쓰든 개발자가 의도한 정확한 A4 레이아웃 보장
- "바로 인쇄하기" 버튼도 → "생성된 PDF를 브라우저 내장 뷰어로 열어서 인쇄창 띄우기"로 구현

**기술 검증 항목**:
- [ ] `@react-pdf/renderer`에서 한글 폰트 깨짐 해결
- [ ] PDF 생성 속도 테스트 (12개월 연간 달력 기준)
- [ ] 모바일 환경에서 PDF 다운로드 동작 확인

#### 출력/다운로드 기능

- **PDF 다운로드 (단일 파일)**
  - A4/A3 기준 PDF
  - 정확한 레이아웃 보장

- **바로 인쇄하기**
  - PDF 생성 후 브라우저 인쇄창 자동 띄우기

- **🆕 프린트 전용 CSS 완벽 분리**
  - `@media print { ... }` 사용
  - 광고 완전 숨김 (중요!)
  - 배경색 제거
  - 불필요한 UI 요소 제거
  - **이거 안 하면 "광고가 달력에 같이 출력돼요" 리뷰로 도배됨**

- **프린트 설정 가이드/팁**
  - 브라우저별 최적 설정 안내 (Chrome/Safari/Firefox)
  - "여백 없음" 설정법
  - A4 용지 크기 확인 알림
  - 양면 인쇄, 잉크 절약 모드 팁

- **🆕 모바일 공유 플로우**
  - 모바일에서 PDF 생성 완료 후
  - "카카오톡으로 공유하기" 버튼 강조
  - PDF 파일 직접 공유 or URL 공유

## 6. 서비스 화면 구조 & 기본 플로우

### 6.1. 주요 페이지

1. **홈 (소개 + 시작 버튼)**
   - 서비스 한 줄 설명
   - "달력 만들기" 주요 CTA 버튼
   - 간단한 예시 이미지(연간/월간 달력 샘플)
   - 🆕 **언어 자동 감지 + 전환**: 브라우저 언어 설정에 따라 자동 한국어/영어
   - 🆕 **"올해", "내년" 퀵 버튼** 제일 상단 배치

2. **달력 생성 페이지**
   - 왼쪽: 설정 패널
     - 🆕 **목적별 프리셋 선택** (기본/습관/식단/공부/근무표)
     - 달력 유형 선택
     - 연도/기간 (올해/내년 퀵 버튼)
     - 국가별 공휴일
     - 종이 크기/방향
     - 스타일 옵션 (Eco 모드, 폰트, 워터마크)
     - 음력/주차 표시 옵션
     - 간단한 메모 입력
   - 오른쪽: 달력 미리보기
     - 옵션 변경 시 미리보기 실시간 갱신
     - 🆕 **오늘 날짜 하이라이트** (연한 노란색 원)
     - 페이지 나누기/넘기기
   - 상단: 🆕 **"이 설정 공유하기" 버튼** (링크 복사, 카카오톡 공유)

3. **출력/다운로드**

## 7. 다른 달력 사이트와의 차별점

   - 🆕 **Eco 모드로 잉크 절약** (실사용자 니즈 정확히 충족)

2. **다국가 공휴일을 한 번에**
   - 한 달력에 여러 나라 휴일을 겹쳐서 볼 수 있음
   - 국가별 색상 커스텀

   - 로그인, 회원가입 없이 바로 사용 가능

   - lostvin 방향과 맞게, UI·문구·레이아웃을 "조용하고 단정한 종이도구"처럼 설계
   - 광고가 화면을 가득 뒤덮지 않도록 밀도 조절

5. **🆕 URL 공유로 바이럴 확산 + 모바일→PC 이어하기**
   - 설정을 URL로 완벽 공유 → SNS/카톡으로 자연스럽게 확산
   - 모바일에서 설정 → PC에서 인쇄 (Stateless 설계)

6. **🆕 목적별 프리셋 (니치 타겟팅)**
